# Per-chunk training-signal sweep over frame_interval (f = 5 / 8 / 10 / 15 / 20)

**Question:** is there meaningful per-chunk training signal for the action branch, and does raising
`frame_interval` (more motion per chunk) help? Original motivation was the FAILED action diagnostic
(Run 001) + the hypothesis that the **high overhead camera mount** dampens how much the image changes.

**Dataset:** the **validation split of `kaushikpraka/wm-smallarea_nav30`** (the derived LeRobot v2.1
dataset at `/workspace/data/lekiwi`, 30 Hz) — 5 held-out episodes. Generated by
`external/nanowm/src/sample/chunk_motion_viz.py` against the Run 001 step-10K checkpoint (used **only**
for its frozen SD-VAE + config — previews any f WITHOUT retraining). Sweep run with `--seed 42
--n-batches 40 --example-mode mixed`.

## Files (per f)
- `chunk_examples.png` — 6 **distinct** chunks (de-duped by episode+time, labelled `ep<traj>@<start>`),
  a mix of **drive** (forward), **rotate**, and **drive+turn** (arc): [GT frame k | GT frame k+f |
  **SD-VAE per-cell ‖Δz‖ map** (32×32)]. The third column is the *latent* change the world model is
  trained to predict (v-pred in SD-VAE space), not a pixel diff; annotated with latL2 = ‖z(k+f)−z(k)‖_F.
- `chunk_distributions.png` — histograms of |Δx|, |Δθ|, pixel-change %, latent-L2, **plus two scatters:
  translation-vs-latentL2 and rotation-vs-latentL2** (the decisive panels).
- `chunk_motion_stats.json` — summary numbers incl. both correlations.

## Read — the camera sees ROTATION, and is nearly blind to TRANSLATION

The high-mount hypothesis is confirmed and **sharpened to a per-component statement**:

- **Translation (Δx) is unobservable in the latent: `corr(Δx, latent-L2) ≈ 0` at every f** (−0.04 …
  +0.04). In the example montage, a full-speed forward chunk (Δx=3.33 cm, Δθ=0°) moves the image only
  **~3%** (latL2 ~27) — the elevated, downward-tilted camera geometrically de-magnifies forward motion,
  and raising f does **not** rescue it (Δx grows 4×, corr stays ~0).
- **Rotation (Δθ) is strongly observable: `corr(Δθ, latent-L2) ≈ 0.64–0.70` at every f.** A pure
  rotation chunk (Δx=0, Δθ≈9.5°) changes the image **~9–10%** (latL2 ~46); an arc (drive+turn) **~12–15%**
  (latL2 ~51–54). Rotation sweeps the *entire* wide field of view, so it dominates the latent.

**Implications.**
1. The roadmap's "retrain at f=8–10" plan is **refuted** for the translation problem — f doesn't move
   `corr(Δx, L2)` off zero. (It does buy a bit more raw motion, but the latent saturates and the
   non-action noise floor grows with the time window.)
2. The Run 001 action-diagnostic failure is really a **translation-observability** failure. The model
   can ground rotation from vision but fundamentally cannot ground forward displacement with this camera.
3. Highest-leverage fixes target *translation* specifically: a **lower / more forward-facing camera**
   (or richer near-field floor texture) for parallax per cm; **auxiliary odometry/pose conditioning**
   for Δx; and lowering the non-action noise floor (exposure/white-balance lock, avoid lossy AV1). See
   `context/open-questions.md` fallbacks.

⚠️ Caveat: `corr(Δx, ·)` with bang-bang Δx is a weak estimator and was noisy on small in-order subsets
(an earlier draw read ~0.2); the seed-42 / n_batches-40 sweep above (≈5–7k chunks/f) is the stable
estimate and lands firmly at ~0. The rotation correlation is robust either way.

## Reproduce / extend
```
python external/nanowm/src/sample/chunk_motion_viz.py \
    --ckpt <any run ckpt> --frame-interval <f> [--seed N] [--n-batches K] \
    [--example-mode mixed|forward|rotate|arc] --out <dir>
```
`--frame-interval` previews any f without retraining; `--seed` varies which scenes are shown;
`--example-mode` picks which motion type to surface (examples are de-duped by episode+time).

See `context/training-runs.md` (Run 001), `context/open-questions.md` ("weak action SNR").
